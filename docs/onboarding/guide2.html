Before you begin

A quick get ready, and what trajecktory is

trajecktory is your own private job search dashboard that runs on your computer. It reads job posts, scores how well they fit you, helps tailor your resume, and keeps track of everything in one place. Claude does the thinking. You stay in control.

Have these ready
  • Guide 1 finished: the Claude app, Git for Windows, and the Code tab all working
  • The trajecktory installer file, named trajecktory-setup-v1.7.23.exe
  • About 20 minutes
Optional: an Anthropic API key, only if you want trajecktory to auto write resumes and cover letters. You can add it now or much later. More on this near the end.
What you will do here
  1. Install trajecktory from the .exe file
  2. Start it from Claude Code (Code mode)
  3. Watch the dashboard open in your browser
  4. Walk the Launchpad from empty to ready
💡 A note on the pictures. A few installer screens here are clean illustrations rather than photos, so your exact windows may look a little different. The buttons and wording to look for are the same.
♥ Two safe truths to keep in mind. First, trajecktory never sends an application or email without you reviewing it. Second, your own files (your tracker, reports, and saved settings) are never overwritten by updates. You can always re edit anything.

Double click the installer and follow its short wizard. Most screens are just a Next button.

Step 1, continued

The one screen worth reading: the API key

Partway through, the installer shows a screen asking for an "Anthropic API key". This sounds technical, but the screen itself tells you exactly what to do, and you are allowed to skip it.

💡 What is an API key? It is a private code that lets trajecktory ask Claude to write things like resumes and cover letters, billed in tiny amounts to your own Anthropic account. It is optional. The scoring and scanning features do not need it. You can add it any time later. We show you how near the end of this guide.
♥ When in doubt, leave it blank. Skipping the key now changes nothing about installing trajecktory or scoring jobs. Click Next, then finish the wizard.
  1. On the API key screen, leave the box empty (unless you already have a key) and click Next.
  2. Click Install, wait a moment, then on the last screen make sure Launch trajecktory now is checked and click Finish.

2

Start the live dashboard from Claude Code

You launch trajecktory from the Code tab you set up in Guide 1. One short phrase does it.
ⓘ First, set Chrome as your default browser. trajecktory opens in whatever browser Windows treats as default. To make it open in Chrome (not Microsoft Edge): open Windows Settings, search default apps, scroll to Web browser, and choose Chrome. You only do this once.
  1. Open the Claude app and click the Code tab (from Guide 1).
  2. Choose Local, click Select folder, and pick your installed trajecktory folder. Paste this path into the folder window: C:\Users\you\trajecktory\trajecktory (replace "you" with your Windows name; yes, "trajecktory" appears twice, the inner folder is the app).
  3. In the message box at the bottom, type Start the live dashboard and press Enter.
  4. trajecktory builds itself and opens the dashboard in your browser. That is it.
♥ No sign-in needed to start. The dashboard opens clean, with no Claude sign-in at launch, and all your setup and data views work right away. (To use the one-click Evaluate and Scan buttons later, there is a one-time sign-in in the sidebar; we cover it under "Using trajecktory.")
↻ Prefer a shortcut? As a fallback, you can also start trajecktory from the trajecktory shortcut the installer placed on your desktop. The Code mode method above is the recommended one.

Meet the Launchpad

A guided checklist that takes you from empty to ready. It does the fiddly parts for you.

When you first open trajecktory, it brings you straight to the Launchpad. A bar at the top, the readiness meter, shows how far along you are. Every setup step is available right away, in any order, and each one you finish turns into a green check.

🛡 You cannot break anything here. The Launchpad only updates your settings. Your applications, reports, and history are never touched, and you can re edit any section later. So click around freely.
✅ About that first item. Preflight check turns green on its own once the engine is ready, which it is on a fresh install. Missing things like your CV or profile are not blockers: they show as friendly "add this below" reminders, and you can start any step right away.
↻ It refreshes on its own. When you switch to Claude Code and come back to the dashboard tab, it re-syncs your data automatically, and a Workflow step's results appear the moment it finishes. You do not need to reload the browser. The "synced ... ago" note in the top bar shows when it last updated.

Your CV · the source of truth for everything
  1. Drag your resume file onto the box, or click browse your desktop to pick it. A Word or PDF file is perfect.

Using trajecktory: the AI steps

Sign in to Claude (one time)

Two of the Workflow steps, Agent Scan and Evaluate Pipeline, use Claude, so they need a one-time sign-in. The free steps do not. The sign-in control lives in the left sidebar, at the top of the Workflow section.

The Workflow section in the sidebar. The sign-in button sits right at the top.
  1. In the sidebar Workflow section, click Sign in to Claude.
  2. A small console window opens and your browser opens a Claude page. If the console asks how to sign in, choose Claude account with subscription.
  3. In the browser, click Authorize.
  4. Important: go back to the console window and answer its last question or two (for example, "Allow external CLAUDE.md imports?" then Yes). You are not signed in from the browser alone.
♥ How you know it worked. The sidebar button changes to "✓ Signed in to Claude". Now Agent Scan and Evaluate Pipeline can run.
⚠ If a step later shows "401 authentication_error: Invalid authentication credentials": that just means the Claude sign-in expired. It is not a bug. Click Sign in to Claude again to refresh it, then retry the step.

Using trajecktory: find and score jobs

The Workflow, one step at a time

In the sidebar, the Workflow section has eight steps. You run them one at a time: click a step, let it finish, then click the next. There is no "run everything" button, on purpose, so you can see each result or error on its own.

The eight steps, in order.
  1. Expand Coverage (optional, free): finds new companies.
  2. API Scan (free): checks the Greenhouse, Ashby, and Lever job boards.
  3. Agent Scan (optional, needs sign-in): Claude searches the web for roles (it shows elapsed time as it runs), and any company it finds with a free job board joins your tracked list, so the free API Scan catches it next time.
  4. Liveness Gate (free): drops dead links.
  5. Evaluate Pipeline (needs sign-in): Claude scores your 5 best-fit postings per run and writes a full report for each. A meter shows "Evaluated 3 of 5 · 1m 20s · ~2m 0s left" with a thin bar. Click Evaluate again for the next 5.
  6. Merge Tracker (free): folds the scored rows into your Pipeline.
  7. Verify Actionable (free): a safety check for dead links.
  8. Health Check (free): a final all-clear.
⚠ After Evaluate Pipeline, click Merge Tracker. While Evaluate runs, the steps below it are disabled and show "waiting on Evaluate Pipeline", then light up when it finishes. Evaluate writes the scored rows; Merge Tracker folds them into the Pipeline tab, so do not skip it. Results appear on their own when a step finishes.
ⓘ "0 new" is normal, not a failure. A scan may say "0 new (of 14,338 found: 14,264 off-title, 28 duplicates, 43 geo-blocked, 3 stale)." Those buckets mean wrong title, the same posting on more than one board, too far away, and expired. To surface more, broaden your target roles or title keywords in Setup. The first full run takes a few minutes; later runs are much faster.

Adding your Anthropic API key later

If you skipped the key during install and later want trajecktory to write tailored resumes, cover letters, or outreach, you can add it any time, right in the dashboard.

  1. Get a key from your Anthropic account at console.anthropic.com (it starts with sk-ant-). Treat it like a password.
  2. In the dashboard, open Setup (the Launchpad), scroll to Optional boosters, and choose AI draft key (optional).
  3. Paste your sk-ant- key and click Save. It takes effect immediately, with no restart.
💡 Prefer to do it during install? You can also paste the key at the installer's optional Anthropic API key step (Step 1). Either route works; the dashboard one is the easiest to do later.
ⓘ Two separate billing buckets. Your Claude sign-in (your Pro or Max plan, with its rolling 5-hour limit) powers onboarding, Agent Scan, and Evaluate, which default to the Sonnet model to go easy on that limit. Your optional API key (pay as you go) powers only the written drafts. They never draw from each other.

If something is not right

Troubleshooting, and where did it go

The most common little snags, and the calm fix for each.

The dashboard did not open. In the Claude Code tab, with your trajecktory folder open, type Start the live dashboard again. The trajecktory shortcut works as a fallback.
"401 authentication_error" on Agent Scan or Evaluate? Your Claude sign-in expired. Click Sign in to Claude in the sidebar again, then retry the step. Not a bug.
The Pipeline looks empty after evaluating. Click Merge Tracker in the sidebar Workflow. Evaluate writes the rows; Merge folds them into the Pipeline tab.
A draft says "add your key". That is the optional API key. See the previous page to add one, or skip it. Scoring still works without it.
"Windows protected your PC". Click More info, then Run anyway. A normal warning for new apps.
The tab disappeared. Look in your browser for the trajecktory tab (localhost), or restart it from Claude Code with Start the live dashboard.
A handoff step will not turn green. Make sure you pasted the instruction into the Claude Code tab (with your trajecktory folder open), let it finish, then click I ran it, refresh status back in the Launchpad.
I see "demo mode" or sample names. That is the practice data. Your real, private setup runs normally; the samples are only there to show how things look.
A "Claude usage or limit pressure" notice. You hit your Claude plan's rolling 5-hour usage limit, usually from heavy use. Not a trajecktory error: wait a bit, then try again.
Want to share what went wrong? Logs live in the app folder's logs\ (agent-runs.1.log to .3.log), plus launch-log.txt and server-out.log in the install folder.
I want to stop trajecktory. Use the Stop trajecktory shortcut. Nothing is lost. Open it again whenever you like.
