Before you begin

A quick get ready, and what trajecktory is

trajecktory is your own private job search dashboard that runs on your computer. It reads job posts, scores how well they fit you, helps tailor your resume, and keeps track of everything in one place. Claude does the thinking. You stay in control.

Have these ready
  • Guide 1 finished: the Claude app installed and signed in (this guide installs Git for you and gets the Code tab working after a restart)
  • The trajecktory installer file, named trajecktory-setup-v1.16.2.exe (a one-time download, about 415 MB)
  • About 20 minutes
Optional: an Anthropic API key, only if you want trajecktory to auto write resumes and cover letters. You can add it now or much later. More on this near the end.
What you will do here
  1. Install trajecktory from the .exe file (it also installs Git for Windows)
  2. Restart your computer once, so the Code tab can see Git
  3. Start trajecktory from Claude Code (Code mode)
  4. Watch the dashboard open in your browser
  5. Walk the Launchpad from empty to ready
💡 A note on the pictures. A few installer screens here are clean illustrations rather than photos, so your exact windows may look a little different. The buttons and wording to look for are the same.
♥ Two safe truths to keep in mind. First, trajecktory never sends an application or email without you reviewing it. Second, your own files (your tracker, reports, and saved settings) are never overwritten by updates. You can always re edit anything.

Install trajecktory from the .exe file

Double click the installer and follow its short wizard. Most screens are just a Next button.
  1. Find trajecktory-setup-v1.16.2.exe (usually in your Downloads folder) and double click it. This is a one-time 415 MB download (it bundles the app, its engine, a browser, and Claude Code). From then on, trajecktory updates itself, so you should not need the big installer again.

  3. The setup wizard opens. Click Next to accept the suggested install location. You do not need to change it.

Step 1, continued

The one screen worth reading: the API key

Partway through, the installer shows a screen asking for an "Anthropic API key". This sounds technical, but the screen itself tells you exactly what to do, and you are allowed to skip it.

💡 What is an API key? It is a private code that can run trajecktory's AI writing (resumes, cover letters, outreach) on your own Anthropic account, billed per token. It is optional and only a faster path: that writing already works on your Claude plan without it. You can add one any time later.
♥ When in doubt, leave it blank. Skipping the key changes nothing about installing trajecktory, scoring jobs, or writing drafts (all of that runs on your Claude plan). Click Next, then finish the wizard.
  1. On the API key screen, leave the box empty (unless you already have a key) and click Next.
  2. Click Install, wait a moment, then on the last screen click Finish.
🔄 Now restart your computer. This step is required, not optional. The installer just installed Git for Windows, which trajecktory and Claude Code both need. Until you restart, the Claude app cannot see Git, so the Code tab will not let you type a prompt, and you cannot start trajecktory. Save any open work and restart your computer once now. When it comes back up, the Code tab is ready and you continue with Step 2 below. (A full restart is the reliable fix. Just quitting and reopening the Claude app is not always enough.)

2

Start the live dashboard from Claude Code

Now that you have restarted, the Code tab can see Git and is ready. You launch trajecktory from it with one short phrase.
ⓘ First, set Chrome as your default browser. trajecktory opens in whatever browser Windows treats as default. To make it open in Chrome (not Microsoft Edge): open Windows Settings, search default apps, scroll to Web browser, and choose Chrome. You only do this once.
  1. After restarting, open the Claude app and click the Code tab. It will now let you type a prompt (before the restart it could not, because Git was not yet visible).
  2. Choose Local, click Select folder, and pick your installed trajecktory folder. Paste this path into the folder window: C:\Users\you\trajecktory\trajecktory (replace "you" with your Windows name; yes, "trajecktory" appears twice, the inner folder is the app).
  3. In the message box at the bottom, type Start the live dashboard and press Enter.
  4. trajecktory builds itself and opens the dashboard in your browser. That is it.
♥ No sign-in needed to start. The dashboard opens clean, with no Claude sign-in at launch, and all your setup and data views work right away. (To use the one-click Evaluate and Scan buttons later, there is a one-time sign-in in the sidebar; we cover it under "Using trajecktory.")
↻ Prefer a shortcut? As a fallback, you can also start trajecktory from the trajecktory shortcut the installer placed on your desktop. The Code mode method above is the recommended one.

Using trajecktory: the AI steps

Sign in to Claude (one time)

The Claude-powered Workflow steps need a one-time sign-in: on the default Claude-plan setup that is Triage and the deep-dive evaluations; with an API key it is Agent Scan and Evaluate. The free steps never need it. The sign-in control lives in the left sidebar, at the top of the Workflow section.

The Workflow section in the sidebar. The sign-in button sits right at the top.
  1. In the sidebar Workflow section, click Sign in to Claude.
  2. A small console window opens and your browser opens a Claude page. If the console asks how to sign in, choose Claude account with subscription.
  3. In the browser, click Authorize.
  4. Important: go back to the console window and answer its last question or two (for example, "Allow external CLAUDE.md imports?" then Yes). You are not signed in from the browser alone.
♥ How you know it worked. The sidebar button changes to "✓ Signed in to Claude". Now Agent Scan and Evaluate Pipeline can run.
⚠ If a step later shows "401 authentication_error: Invalid authentication credentials": that just means the Claude sign-in expired. It is not a bug. Click Sign in to Claude again to refresh it, then retry the step.

Using trajecktory: find and score jobs

The Workflow, one step at a time

In the sidebar, the Workflow section lists the steps that find and score jobs. You run them one at a time: click a step, let it finish, then click the next (there is no "run everything" button, on purpose). What you see depends on your setup: by default, on your Claude plan, you get a short flow built around Triage. Add an API key and bill to it, and the sidebar switches to the fuller power flow (see the note below).

The sidebar Workflow, as it looks on the default Claude-plan setup.
  1. API Scan (free): checks the Greenhouse, Ashby, and Lever job boards for your tracked companies.
  2. Triage (needs sign-in): Claude does a cheap first pass and scores your best matches. They appear as a scored list right below the steps.
  3. Deep dive the strong ones: on any scored card, click Deep dive to write a full report for that role. (You can also paste a job link or JD into the box below to evaluate it on the spot.)
  4. Merge Tracker (free): folds the scored rows into your Pipeline tab.
  5. Verify Actionable (free): a safety check that drops dead links.
  6. Health Check (free): a final all-clear.
⚡ The power flow (with an API key). If you save a key and set billing to it in Models & cost, the sidebar swaps Triage for the full pipeline: API Scan, Agent Scan (Claude web search), Liveness Gate, Evaluate (Batch) (full reports for your best-fit postings; a meter shows "Evaluated 3 of 5"), Merge Tracker, Verify Actionable, Health Check, and Expand Coverage. Same idea: run each in turn.

What the key does and does not do. It unlocks that longer step list and a bigger Evaluate batch. It is not what pays for these steps: scanning and evaluating still run on your Claude plan, and your key is only a fallback for the rare moment your plan sign-in is unavailable. The app says so on the sidebar itself. Only Insights and Drafts reliably bill the key.
⚠ Always finish with Merge Tracker. Scoring a role (a Triage deep-dive, or an Evaluate batch) writes the report; Merge Tracker is what folds those rows into your Pipeline tab, so do not skip it. While a scoring step runs, the steps below it wait their turn, then light up when it finishes. Results appear on their own as each step completes.
ⓘ "0 new" is normal, not a failure. A scan may say "0 new (of 14,338 found: 14,264 off-title, 28 duplicates, 43 geo-blocked, 3 stale)." Those buckets mean wrong title, the same posting on more than one board, too far away, and expired. To surface more, broaden your target roles or title keywords in Setup. The first full run takes a few minutes; later runs are much faster.

Using trajecktory: your daily rhythm

The Today tab, your plan for right now

A job search goes better with a routine than with willpower, so Today gives you one. On the left are your time blocks for today, each with a built-in focus timer. On the right is your to-do list. The little number beside Today in the menu counts the blocks you have not finished yet plus anything overdue, so your goal each day is simply to get it to disappear.

Blocks, and the timer
Click ▶ Start on a block and a focus timer runs. While it is going you get Pause (which becomes Resume), Skip, and Stop. Two small switches, sound and notify, chime or pop up a desktop notice when a block ends. Tick a block's box to mark it done.
To-dos
Type into Add a to-do and press Enter, then set High, Med, or Low, and a due date if you want one. The chips above the list narrow it down: Open, Today, Overdue, Done, All.
📅 Set your week once, in the Schedule sub-tab. At the top of the tab, click Schedule to open Weekly schedule. Each row is one block: name it, pick the days with the quick buttons (MWF, TT, Weekdays, Every day), set a start time and a length, then click Save schedule. + Add block adds another row, and Archive retires a block you have outgrown without losing its history. trajecktory starts you off with four sensible blocks, so you can just rename those.
♥ Turning a job into a task. Open any role in the Pipeline, go to its Notes tab, and use Add a to-do. It appears on this tab tagged with that company, so "prep for their screen" stops living in your head. A small Current streak counter also tracks the days you finished everything, if that is the sort of thing that keeps you going.

Using trajecktory: preparing to interview

The Interview tab, your prep and your live board

This is where trajecktory helps you get ready for an actual conversation. A company shows up here once you move it to Phone Screen or beyond in the Pipeline and it has prep notes written. Until then the tab politely reads No interviews on deck, which is exactly right on your first day.

Prep
Your research and your stories, written out in full: what to say, what to avoid, and what to ask them. A Sections list down the side jumps you around a long document.
Live
A stripped-back board for during the call. You click a prompt and its answer appears in big type, positioned just under your webcam so your eyes stay up.
💬 Adding a round uses the same handoff you already know. Click + New round, type the round number and optionally a short description (final-loop, panel, hm), and trajecktory hands you a ready-made instruction. Paste it into the Claude Code tab exactly like the Launchpad steps on page 7, then come back and refresh. Nothing is written until you run it, so clicking the button on its own is always safe.
ⓘ Two buttons worth knowing before an interview. Present throws the board full screen over everything else, and Esc brings you back. Print offers either the Full prep document or a Cram sheet, a one or two page version for the ten minutes beforehand. Both print or save as a PDF. (Do not confuse this tab with the Interview tab inside a job's report, which holds the likely questions for that one role.)

Keeping it current

Keeping trajecktory up to date

After that first install, trajecktory updates itself from inside the dashboard. There is nothing to download by hand, and nothing to do in Claude Desktop.

  1. When you open the dashboard, trajecktory checks for a newer version (this needs an internet connection).
  2. If one is ready, a banner appears across the top: Update available: v1.16.1 to v1.16.2, with a What's new link (the changelog) and an Update now button.
  3. Click Update now. trajecktory downloads the update, restarts itself, and reloads the page on its own. That is the whole process.
♥ Your data is never touched. Updates only replace the program files. Your CV, profile, application tracker, reports, and settings stay exactly as they are, and trajecktory keeps a rollback in case anything ever goes wrong.
ⓘ Check which version you are on. The dashboard sidebar shows the running version, for example v1.16.2, so you can confirm an update applied.
📥 Already on an older build (1.7.23 or earlier)? Those cannot update themselves, so install this version once from the installer. After that single manual install, every future update is one click.
⚠ One rare exception. Now and then an update needs a brand-new installer (when the underlying engine changes). When that happens, the banner tells you to download the latest setup file instead of updating in place. That is the only time you would need the big installer again.

If something is not right

Troubleshooting, and where did it go

The most common little snags, and the calm fix for each.

The dashboard did not open. In the Claude Code tab, with your trajecktory folder open, type Start the live dashboard again. The trajecktory shortcut works as a fallback. (If the Code tab will not let you type, you have not restarted since installing trajecktory. Restart once, then try again.)
"401 authentication_error" on Agent Scan or Evaluate? Your Claude sign-in expired. Click Sign in to Claude in the sidebar again, then retry the step. Not a bug.
The Pipeline looks empty after evaluating. Click Merge Tracker in the sidebar Workflow. Evaluate writes the rows; Merge folds them into the Pipeline tab.
A draft says "add your key". That is the optional API key. See the previous page to add one, or skip it. Scoring still works without it.
"Windows protected your PC". Click More info, then Run anyway. A normal warning for new apps.
Setup says "Git for Windows is not installed," or the Code tab will not let me type. trajecktory installs Git for you, but the Claude app only sees it after a restart. Restart your computer once, then reopen the Code tab and continue. This is normal on a fresh install.
